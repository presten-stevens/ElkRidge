<objective>
Add outbound message retry so that when BlueBubbles is temporarily unreachable, POST /send messages are queued and retried with exponential backoff instead of being silently lost.

Purpose: Eliminates the single biggest data loss risk in the outbound flow -- BB downtime currently means dropped messages with only a log entry.
Output: OutboundRetryQueue service, modified send route catch block, server wiring, full test coverage.
</objective>
