    - When ALERT_WEBHOOK_URL is not set: log warning, skip alert POST (D-10)
    - Alert POST payload matches AlertPayload shape (D-09)
    - Alert POST failure: log error, do NOT throw (fire-and-forget)

    - Calling initHealthMonitor twice clears old timer first (no timer leak)
    - lastChecked on HealthResponse is updated after each successful poll
  </behavior>

    2. Create src/services/health-monitor.ts (TDD green):
       - Module-level state: timer, consecutiveFailures = 0, alertFired = false, lastChecked: string | null = null
       - Export function initHealthMonitor(client: BlueBubblesClient): void
         - If timer exists, clearInterval(timer) first (prevent leak per Pitfall 5)
         - timer = setInterval(() => pollHealth(client), env.HEALTH_POLL_INTERVAL_MS)
         - timer.unref() (per D-06)
       - Export function shutdownHealthMonitor(): void
         - clearInterval(timer), reset state
       - Private async function pollHealth(client: BlueBubblesClient): void
         - const result = await checkHealth(client)
         - Update lastChecked = new Date().toISOString() if result.status !== 'down'
         - If result.status === 'healthy': reset consecutiveFailures = 0, alertFired = false, log debug
         - If result.status === 'degraded' or 'down': increment consecutiveFailures
           - If consecutiveFailures === env.ALERT_AFTER_FAILURES && !alertFired:
             - Determine service: result.status === 'down' ? 'bluebubbles' : 'imessage'
             - Build AlertPayload per D-09: { type: 'downtime_alert', service, status: result.status, message: descriptive string, timestamp: ISO }
             - Call sendAlert(payload)
             - Set alertFired = true
       - Private async function sendAlert(payload: AlertPayload): void
         - If !env.ALERT_WEBHOOK_URL: logger.warn('ALERT_WEBHOOK_URL not configured, skipping alert'), return (per D-10)
         - try { fetch(env.ALERT_WEBHOOK_URL, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(payload), signal: AbortSignal.timeout(10_000) }) }
         - On !res.ok: logger.error with status
         - On catch: logger.error with message (fire-and-forget, no throw)
       - Export function getLastChecked(): string | null -- for health route to include lastChecked
       - Run tests -- they MUST pass (green)
