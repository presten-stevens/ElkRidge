<objective>
Add file-based persistence to the RetryQueue so webhook retry entries survive process restarts.

Purpose: Currently, if the service restarts while CRM webhook deliveries are queued for retry, those entries are lost. This phase ensures zero message loss across restarts by persisting queue state to `data/retry-queue.json` using the same atomic temp-file-then-rename pattern as `sync-state.ts`.

Output: Modified `RetryQueue` class with `init()` (load from disk) and `persist()` (save to disk) methods, updated `webhook-relay.ts` with async `initRelay`/`shutdownRelay`, and updated `server.ts` to await the async init.
</objective>
