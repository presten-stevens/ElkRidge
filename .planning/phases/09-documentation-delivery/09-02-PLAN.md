**Part B: docs/HANDOFF.md (per D-08)**

Title: "Source Code Handoff"

Sections:
1. **Repository Structure**: Tree showing top-level dirs and their purpose:
   - src/ — Application source (TypeScript)
   - src/routes/ — Express route handlers (send, conversations, health)
   - src/services/ — Business logic (BlueBubbles client, rate limiter, health, webhook relay, retry queue)
   - src/middleware/ — Express middleware (auth, error handler, logger)
   - src/types/ — TypeScript types and error codes
   - src/config/ — Environment configuration
   - deploy/ — Deployment configs (nginx, PM2 startup)
   - docs/ — Documentation
   - dist/ — Compiled output (generated by npm run build)

2. **Build, Test, Run**:
   - Install: `npm install`
   - Build: `npm run build`
   - Test: `npm test`
   - Dev mode: `npm run dev`
   - Production: `pm2 start ecosystem.config.js`
   - Lint: `npm run lint`

3. **Key Architecture Decisions** (summarize from STATE.md decisions):
   - Fire-and-forget send: POST /send returns immediately with "queued" status, actual send happens async with jitter
   - Singleton services: BlueBubbles client, rate limiter, dedup buffer use factory pattern (getRateLimiter, getBBClient)
   - Webhook relay: Socket.IO listener -> dedup -> relay to CRM URL, with retry queue for failures
   - Backfill on reconnect: queries BB for messages since last_synced_at on startup/reconnect
   - Health monitoring: periodic polling with alert state machine (consecutive failures trigger single alert)
   - Express binds 127.0.0.1 only, nginx handles all public traffic
   - No database: only local JSON file for last_synced_at
